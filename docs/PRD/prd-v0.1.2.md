# PRD v0.1.2 - 資料驗證修復與使用體驗優化

**發布日期**: 2025-10-14
**版本**: v0.1.2
**類型**: Bug Fix & UX Enhancement

## 📋 更新摘要

本次更新主要解決 Firestore 資料驗證錯誤，並優化後台管理介面的使用體驗。修復了課程資料中 `undefined` 值導致的資料庫寫入失敗問題，同時改善了表單輸入的可讀性與網站元資料。

---

## 🐛 Bug 修復

### 1. Firestore Undefined 值驗證錯誤
**問題描述**:
在課程管理後台建立或編輯課程時，Firestore 拋出錯誤：
```
Value for argument "data" is not a valid Firestore document.
Cannot use "undefined" as a Firestore value (found in field "modules.X.description")
```

**根本原因**:
- `lib/admin-data.ts` 中的資料正規化函數會將空字串轉換為 `undefined`
- 涉及的函數：`normalizeSyllabusArray()`, `normalizeModuleArray()`, `normalizeCourseInput()`
- 受影響的欄位：`modules[].description`, `lessons[].description`, `lessons[].videoUrl`

**解決方案**:
- 修改所有正規化函數，將 `undefined` 改為空字串 `""`
- 在 API 層加入 `removeUndefined()` 工具函數作為額外防護

**修改檔案**:
- `lib/admin-data.ts` (lines 217-218, 221-224, 275-276, 280-282, 311-312, 767)
- `app/api/admin/courses/route.ts`
- `app/api/admin/courses/[courseId]/route.ts`

### 2. Next.js 15 Params 型別錯誤
**問題描述**:
訂單詳情頁面出現 404 錯誤，控制台顯示：
```
Route used `params.orderId`. `params` should be awaited before using its properties.
```

**根本原因**:
Next.js 15 要求動態路由的 `params` 必須是 `Promise` 型別

**解決方案**:
```typescript
// 修改前
export default async function Page({ params }: { params: { orderId: string } }) {
  const order = await getOrderById(params.orderId);
}

// 修改後
export default async function Page({ params }: { params: Promise<{ orderId: string }> }) {
  const resolvedParams = await params;
  const order = await getOrderById(resolvedParams.orderId);
}
```

**修改檔案**:
- `app/admin/orders/[orderId]/page.tsx`
- `app/orders/[orderId]/page.tsx`

---

### 3. 空白縮圖導致的圖片載入錯誤
**問題描述**:
課程列表頁面出現錯誤：
```
An empty string ("") was passed to the src attribute.
This may cause the browser to download the whole page again.
```

**根本原因**:
修復 Firestore undefined 問題後，`thumbnail` 欄位變成空字串，而 Next.js Image 組件不接受空字串

**解決方案**:
```typescript
<Image
  src={course.thumbnail || "/placeholder-course.jpg"}
  alt={course.title}
  fill
/>
```

**修改檔案**:
- `components/courses/CourseCatalog.tsx`

## 🔧 技術細節

### 資料正規化邏輯改進
```typescript
// 修改前
description:
  typeof data.description === "string"
    ? data.description.trim() || undefined
    : undefined

// 修改後
description:
  typeof data.description === "string"
    ? data.description.trim() || ""
    : ""
```

### API 防護機制
新增 `removeUndefined()` 工具函數：
```typescript
function removeUndefined<T>(obj: T): T {
  const jsonString = JSON.stringify(obj, (key, value) => {
    return value === undefined ? null : value;
  });
  return JSON.parse(jsonString) as T;
}
```
